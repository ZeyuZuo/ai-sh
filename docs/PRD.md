# PRD — ai-sh：自然语言命令行助手

**版本** 0.1  
**状态** 草稿  
**最后更新** 2026-06-09

---

## 1. 背景与目标

### 1.1 问题陈述

命令行是开发者和运维人员最高效的工具，但记忆大量命令语法是持续的认知负担。用户知道自己想做什么，却不知道或记不住对应的 shell 命令——尤其是涉及复杂参数、管道组合、或不常用工具时。

现有解法（man page、搜索引擎、cheatsheet）都要打断工作流，离开终端，再回来复制粘贴。

### 1.2 产品目标

构建一个命令行工具 `ai-sh`，让用户用自然语言描述意图，AI 翻译成可执行的 shell 命令，并在用户确认后在原终端执行。全程不离开命令行环境。

### 1.3 不在范围内（v0.1）

- GUI 或 Web 界面
- 自动化脚本生成（多步骤 workflow）
- 云端命令历史同步
- 团队协作功能
- 插件系统

---

## 2. 目标用户

**主要用户：** 有一定命令行基础、但不是 shell 专家的开发者。知道自己想做什么，偶尔需要翻文档的那类人。

**次要用户：** 运维 / DevOps 工程师，需要快速构造不熟悉系统（如 Kubernetes、AWS CLI）的命令。

**不针对：** 完全不懂命令行的用户（没有足够的上下文判断 AI 输出是否正确）。

---

## 3. 核心用户场景

### 场景 A：单次查询（最高频）

```
$ ai "找出当前目录下超过 100MB 的文件"
```

用户得到命令、解释和风险等级，选择执行或取消。

### 场景 B：REPL 连续对话

```
$ ai-sh
ai> 找出最近一天修改的 python 文件
ai> 把刚才的结果只保留 src/ 目录下的
ai> 用 wc -l 统计一下行数
```

用户在一个会话里追问，AI 保持上下文，理解「刚才」「上一步」这类指代。

### 场景 C：管道输入

```
$ git diff | ai "帮我总结这次改动"
$ cat error.log | ai "这个报错是什么意思，怎么修"
```

从 stdin 读取内容作为上下文附加到请求中。

### 场景 D：撤销 / 追问

```
ai> 刚才那个命令加上 --dry-run 参数
ai> 改成递归处理子目录
```

基于上一条命令的修改，不需要重新描述整个意图。

## 4. 功能需求

### 4.1 命令生成

- AI 根据用户输入和环境上下文生成 shell 命令
- 每条命令附带：
  - 自然语言解释（说清楚命令做什么）
  - 风险等级：`safe` / `caution` / `danger`
  - 风险原因（仅 caution / danger 时展示）
  - 备选命令（可选，当存在更简单或更安全的写法时提供）
- 当用户意图模糊时，AI 应询问澄清而不是猜测

### 4.2 环境上下文感知

生成命令前，自动收集以下上下文注入 prompt：

- 当前工作目录（`cwd`）
- 所用 shell 类型（`bash` / `zsh` / `fish`）
- 操作系统（`macOS` / `Linux` 及发行版）
- 当前用户名
- PATH 中检测到的关键工具（`git`、`docker`、`kubectl`、`python3` 等）
- 最近 N 条命令历史（REPL 模式下为对话历史）

### 4.3 安全机制

**本地硬拦截（不经过 AI）：**

在将命令展示给用户之前，用正则匹配以下模式，命中则直接拒绝执行并说明原因：

- `rm -rf /` 及变体
- `rm -rf ~`
- `mkfs.*`（格式化磁盘）
- `dd if=... of=/dev/sd*`（直写磁盘设备）
- `> /dev/sd*`
- `: (){ :|: & }; :`（fork bomb）
- `base64 -d ... | bash` 及类似远程执行管道
- `chmod -R 777 /`

**AI 侧风险标记：**

要求 AI 在响应中返回 `risk_level` 字段。`safe` 级别且未命中本地硬拦截时自动执行；`caution` 级别需用户明确确认一次；`danger` 级别命令即使通过本地检测也不执行（展示警告后终止）。

**执行前确认：**

`caution` 命令在执行前必须经过用户确认。`safe` 命令默认自动执行；用户可通过 `--dry-run` 只查看不执行。

### 4.4 交互确认流程

`caution` 命令展示后，提供三个选项：

- `[y]` 执行
- `[e]` 在编辑器中修改后执行
- `[n]` 取消（默认）

`danger` 级别命令不进入确认流程，直接拒绝。

### 4.5 多轮对话与历史

- REPL 模式下维护完整对话历史，支持追问和修改
- 执行结果（stdout / stderr 的摘要）回写入对话历史，让后续追问更准确
- 命令历史持久化到本地（`~/.ai-sh/history.json`），重启后可复用
- 单次模式（`ai "..."`）不维护跨调用历史，每次独立

### 4.6 配置

通过 `~/.ai-sh/config.toml` 配置：

```toml
[api]
base_url = "https://api.siliconflow.cn/v1"   # 第一版默认 SiliconFlow OpenAI 兼容接口
model = "deepseek-ai/DeepSeek-V3.2"
api_key = ""   # 也可通过环境变量 SILICONFLOW_API 覆盖

[behavior]
default_confirm = "n"       # 默认选项（安全起见默认取消）
history_limit = 50          # 保留最近多少条历史
context_commands = 5        # 注入 prompt 的历史命令条数
language = "zh"             # 响应语言：zh / en / auto

[safety]
hard_block_enabled = true   # 本地正则拦截开关（建议保持 true）
```

第一版默认使用 SiliconFlow 的 OpenAI 兼容 API；后续仍保留 `base_url` 配置项，以便切换到其他 OpenAI 兼容服务。

## 5. 非功能需求

### 5.1 性能

- 单次请求从输入到展示命令的延迟 ≤ 3 秒（取决于 API，本地处理部分 ≤ 50ms）
- 启动时间 ≤ 200ms（不影响日常终端使用体验）

### 5.2 兼容性

- **操作系统：** macOS 12+，主流 Linux 发行版（Ubuntu 20.04+，Debian 11+，Arch）
- **Shell：** bash、zsh、fish（命令语法生成适配各 shell 差异）
- **Python：** 3.10+
- **Windows：** v0.1 不支持（WSL 用户可用 Linux 路径）

### 5.3 可靠性

- API 调用失败时给出清晰错误提示，不 crash
- 网络超时后提示用户重试，不挂起终端
- 本地历史文件损坏时优雅降级（清空重建，不影响主功能）

### 5.4 安全

- API Key 不写入命令历史、不打印到终端
- 不收集用户命令数据到任何远端（除调用用户自己配置的 API）
- 历史文件权限设为 `600`
